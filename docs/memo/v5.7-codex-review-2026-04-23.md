**3. 新規 Must-Fix**

1. **MF-v5.7-1: `state_fallback()` の関数契約コメントが古い**
   `docs/architecture-v5.md:2000-2002` は「全 attempts failure なら `Action.WAIT`」と書いているが、v5.7 の modal failure は `WAIT_FOR_SUPERVISOR`。実装者が docstring を優先した場合に modal deadlock 修正を巻き戻す危険がある。

2. **MF-v5.7-2: `WAIT_FOR_SUPERVISOR` の payload / reason / nullable modal が未定義**
   `modal is None` branch は妥当な防御だが、`docs/architecture-v5.md:2126-2129` と Appendix D は `supervisor_request.modal` に現在 modal state がある前提で、例も level-up 固定。`modal_desync` 時は `modal: null` を許すのか、代替 snapshot を含めるのかを明記する必要あり。

他 branch の `Action.WAIT` は意味論的に適切。combat last-resort は安全 no-op、navigation `auto_act_active` は AutoAct 継続、idle/navigation failure は非 modal なので supervisor ではなく deterministic wait でよい。

**6. 実装者が最初につまずくポイント Top 7**

1. **CoQ mod packaging**: `.csproj` / DLL ではなく `Mods/LLMOfQud/*.cs` を CoQ が Roslyn compile する点。
2. **begin-turn callback の重複登録**: `RegisterOnBeginPlayerTurnCallback()` は重複 guard なし。
3. **modal fallback の supervisor payload**: `WAIT_FOR_SUPERVISOR` が bare enum に見えるが、実装には reason/modal/diagnostic が必要。
4. **new popup 操作スレッド**: `PopupMessage.OnSelect` / `OnActivateCommand` は `uiQueue`、gameQueue 直投げは freeze 要因。
5. **`BackgroundClicked()` の null deref**: cancel item lookup を自前で null-safe にする必要あり。
6. **ScreenBuffer 読み取り**: `TextConsole.BufferCS` lock と `ViewTag` を見ないと map と popup を取り違える。
7. **`turn_start` と command loop の関係**: player speed/energy により 1 turn_start = 1 command ではない。
